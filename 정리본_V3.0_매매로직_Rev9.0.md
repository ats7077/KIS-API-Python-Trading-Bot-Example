1. 전략 범위와 문서 사용 원칙

• 범위: 본 문서는 V3.0 전략의 주문 생성·체결 반영·상태 전이 규칙을 자동매매 구현 관점에서 정의한다.

• 우선순위: 예시 숫자보다 일반식과 상태 규칙을 우선하며, 문서의 모든 주문은 “어떤 조건에서 어떤 주문세트를 생성하는가”로 기술한다.

• 주문 구조 원칙: 일일 주문세트의 기본 구조는 “기본 주문세트 + 줍줍매수 사다리”이다. 큰수매수는 별도 전략이 아니라, 브로커 가격 괴리 규칙으로 상단 매수부가 거부되거나 거부가 예상될 때 기본 주문세트의 상단 매수부만 재구성하는 규칙이다.

• 체결 기준: 주문 생성은 전일 확정 체결을 반영한 뒤 수행하며, 당일 중 발생한 체결 결과는 다음 주문 생성 시점의 상태값으로 반영한다.

4. 첫 진입과 일반모드 체결 규칙

4.1 첫 진입 (T = 0)

• 실행 조건: 이전 사이클이 종료되어 H = 0이고, 다음 거래일 새 사이클을 시작해야 할 때만 발동한다.

• 기본 가격: 전일 종가 대비 상단 버퍼(+15% 기본)를 적용한 체결 우선 LOC 가격을 사용한다.

• 기본 수량: 해당 주문세트 예산 전체를 가능한 한 많이 소진하는 정수 수량을 계산한다.

• 동반 주문: 첫 진입 기본 주문과 동시에 하단에는 줍줍매수 사다리를 함께 배치한다. 줍줍매수는 특수 옵션이 아니라 기본 주문세트의 동반 사다리다.

• 매도 주문: 첫 진입 시점에는 보유수량이 없으므로 매도 주문은 생성하지 않는다.

5. 큰수매수 공통 알고리즘

큰수매수는 버전별로 다른 전략이 아니라, 일반모드 또는 특수모드 기본 주문세트가 브로커 가격 괴리 규칙에 의해 거부될 때 적용하는 공통 재구성 알고리즘이다.

<!-- table_11 -->
| 구분 | 규칙 |
| --- | --- |
| 발동 조건 | 현재가와 기본 LOC 주문가가 크게 벌어져 접수 거부 또는 거부 예상 상태가 확인될 때 |
| 기준점 설정 | 현재가보다 충분히 높은 버퍼 가격을 큰수 기준점 K로 정한다. K는 “해당 가격 이하에서 실제 매수가 진행되게 만드는 값”으로 선택한다. |
| 상단 물량 합산 | 기존 주문세트 중 K 이상 가격에서 체결될 예정이던 모든 매수 물량을 하나로 합산하여 K에 단일 LOC 매수한다. |
| 하단 사다리 유지 | 큰수 기준점 K 아래의 하단 1주 주문과 기본 주문세트에 동반된 줍줍매수 사다리는 그대로 유지한다. 큰수매수는 상단 매수부만 압축하는 재구성 규칙이다. |
| 매도 처리 | 큰수매수는 매수 측 재구성 규칙이다. 매도 주문은 원칙적으로 기존 구조를 유지하되, 브로커가 매도까지 거부하면 매도 주문도 별도 안전정책에 따라 생략 또는 조정한다. |

| 큰수매수 해석 고정• 큰수매수의 목적은 숫자 예시 재현이 아니라 상단 체결 보장이다.• 큰수 기준점 이상 가격에 있던 물량은 모두 합산하여 단일 LOC로 재구성한다.• 큰수매수는 매수측 주문구조를 바꾸는 로직이며, 매도 규칙을 새로 만드는 로직이 아니다. |

6. 줍줍매수 공통 알고리즘

줍줍매수는 기본 주문세트와 함께 항상 생성하는 하단 예산 소진 사다리다. 주문을 내는 시점에는 당일 하락을 예측할 수 없으므로, 기본 주문세트와 동시에 하단 사다리를 미리 배치하여 해당 모드의 유효 예산을 가능한 한 넓은 가격 구간에 소진하도록 설계한다. 본 문서에서는 본질을 “기본 주문세트에 동반되는 하단 확장 규칙”으로 정의하고, 단수는 필요한 범위까지 확장 가능하다고 본다.

<!-- table_13 -->
| 항목 | 규칙 |
| --- | --- |
| 입력값 | 해당 모드의 유효 1회 매수시도액 M_t, 기본 주문세트로 계산된 기준 체결수량 Q0. |
| 가격식 | J_n = M_t / (Q0 + n), n = 1, 2, 3, ... |
| 발행 방식 | 각 n에 대해 1주 LOC 매수 주문을 생성한다. 가격은 센트 2자리 기준 정규화 후 하단 가격 사다리로 배치한다. 이 사다리는 기본 주문세트와 항상 함께 생성된다. |
| 중단 조건 | 예산이 허용하는 범위까지 단수를 확장한다. “6단”은 운영상 예시일 수 있으나, 문서 본질은 예산 소진을 위한 하단 사다리 확장 규칙이다. |
| 예시 프로파일 | 큰수매수로 상단 매수부를 합산한 경우에도 줍줍매수는 유지한다. 필요한 경우 합산 후의 Q0를 기준으로 단수를 다시 산정한다. |

7. V3.0 전용 자금 규칙: 반복리 / 세이브 / 쿼터손절

<!-- table_14 -->
| 항목 | 규칙 |
| --- | --- |
| 실현손익 계산 | 쿼터매도 체결마다 해당 체결분의 실현손익 G_realized를 계산한다. |
| 양수 손익 반영 | G_realized > 0이면 절반은 다음 거래일 매수시도액 증가 재원으로, 나머지 절반은 Save로 적립한다. |
| 손실 또는 0 | G_realized ≤ 0이면 M_t는 증가시키지 않고 직전 값을 유지한다. Save도 증가시키지 않는다. |
| 반영 시점 | 당일 장중 즉시 반영하지 않고, 당일 마감 후 다음 거래일 주문 생성 단계에서만 반영한다. |
| 전량매도 손익 | 사이클 종료 손익은 쿼터매도 반복리 계산에 섞지 않는다. 다음 사이클 시작 시 총 원금 재분할로만 반영한다. |
| 운용 중 원금 증액 | 사이클 진행 중 외부 증액은 허용하지 않는다. 원금 재정의는 사이클 종료 후 새 시작 시점에만 가능하다. |

<!-- table_15 -->
| 항목 | 규칙 |
| --- | --- |
| 진입 조건 | a - 1 < T ≤ a 인 구간에 처음 진입한 거래일 |
| 진입일 매수 | 진입일 매수는 없음. 일반모드 매수 및 일반모드 줍줍과 병행하지 않는다. |
| 진입일 매도 | 당일 1/4은 MOC 매도, 나머지 3/4는 A × (1 + R_q/100) 지정가 매도를 배치한다. |
| 진행일 자금 | 진입일 이후에는 후반전과 동일한 기본 주문세트 + 동반 줍줍매수 사다리를 사용한다. 필요하면 세이브 자금을 추가 투입하며, 상단 매수부는 큰수매수로 재구성 가능하다. |
| 진행일 주문 | 진행일 매수는 후반전과 동일하게 S_buy 중심으로 생성하되, 동반 줍줍매수 사다리를 함께 낸다. |
| 모드 중 수익 발생 | 진행일 매도는 1/4 LOC 매도와 나머지 3/4의 +R_q 지정가 매도를 유지한다. |
| 재고갈 | Save까지 소진되어 다시 매수 여력이 고갈되면 진입일과 같은 1/4 MOC + 3/4 지정가 구조로 현금을 확보한다. |

<!-- table_16 -->
| V3.0 자금 철학• V3.0은 순매수누적액 B_net과 가변 1회 매수시도액 M_t를 함께 해석하는 버전이다.• 쿼터매도는 B_sell 증가를 통해 T 해석을 바꾸고, 동시에 양의 실현손익은 다음 거래일 M_t를 키울 수 있다.• 세이브 자금은 운영 중 원금 증액의 대체가 아니라 전략 내부에서 적립된 별도 완충자금으로 취급한다. |

11. 문서 사용 시 주의사항

• 백테스트 해석: 본 문서는 실전 주문 생성 명세서다. 백테스트 엔진은 브로커의 LOC/MOC/지정가 체결 모델을 별도로 구현해야 한다.

• 수치 예시의 지위: 예시 숫자는 문서 중심이 아니다. 실제 구현은 일반식과 상태 규칙을 우선하며, 숫자 예시는 검산 보조 수단으로만 사용한다.

• 버전 차이 유지: 형식은 V4.0형으로 통일했지만, V2.2와 V3.0의 자금 철학과 특수모드 구조 차이는 그대로 유지한다.
