1. 주요 용어 및 기호 정의

아래 기호는 본 문서 전체에서 동일 의미로 사용합니다. 수식은 원문 예시를 일반화한 형태이며, 금액·가격·상태·수량 계열을 서로 겹치지 않도록 분리했습니다.

<!-- table_3 -->
| 용어 | 기호 | 정의 | 수식 |
| --- | --- | --- | --- |
| 원금 | P | 무한매수법에 단독으로 할당하여 사용하는 총 투자금입니다. 시작 시점에 고정되며, 사이클 중 외부 자금 추가 없이 운용하는 것을 원칙으로 합니다. | - |
| 매매종목 | Ticker | 주요 적용 종목입니다. 오피셜 기준 대표 예시는 TQQQ와 SOXL이며, 변동성이 큰 3배 레버리지 ETF를 주 대상으로 합니다. | - |
| 목표수익률 | R | 최종매도 지정가와 별% 계산의 기준이 되는 목표 수익률입니다. 대표값은 TQQQ 15, SOXL 20입니다. | - |
| 현재평단가 | A | 현재 보유수량 기준 총 매수원가를 현재 보유수량으로 나눈 가격입니다. 일반모드와 리버스모드 모두 별지점과 종료조건 계산의 기준이 됩니다. | A = 총 매수원가 / 보유수량 |
| 분할 | a | 전체 원금을 몇 회로 나누어 운용할지 나타내는 총 분할 수입니다. 원문 예시는 20·40분할 중심이지만, 자동매매 관점에서는 분할 수가 달라도 동일 원리로 일반화합니다. | - |
| 잔금 | C | 전일 체결을 반영한 뒤 즉시 매수에 사용할 수 있는 실제 가용현금입니다. 일반모드 1회매수시도액과 리버스모드 1회매수시도액 계산의 기준입니다. | C_new = C_old - 당일 체결 매수금액 합계 + 당일 체결 매도금액 합계 |
| 회차 | T | 현재 사이클의 진행 정도를 나타내는 상태값입니다. 일반모드에서는 1회 매수 체결 시 +1, 절반 매수 체결 시 +0.5, 쿼터매도 체결 시 직전 T × 0.75를 적용합니다. | - |
| 별% | Star(T) | 현재 T와 목표수익률에 따라 계산되는 별지점용 퍼센트입니다. 분할 수가 달라도 동일 원리로 일반화합니다. | Star(T) = R × (1 - 2T/a) |
| 별지점가격 | S | 현재 평단가 A와 별%를 반영해 계산한 별지점 원시 가격입니다. 매수·매도 기준점의 계산 출발점으로 사용합니다. | S = A × (1 + Star(T)/100) |
| 별지점 매수점 | S_buy | 정규화 별지점에서 0.01달러를 뺀 LOC 매수 기준 가격입니다. | S_buy = S_norm - 0.01 |
| 별지점 매도점 | S_sell | 정규화 별지점 그대로 사용하는 LOC 매도 기준 가격입니다. | S_sell = S_norm |
| 전반전 | - | 일반모드에서 평단가 LOC와 별지점 LOC를 함께 사용하는 구간입니다. | T < a/2 |
| 후반전 | - | 일반모드에서 평단가 LOC를 제거하고 별지점 LOC 단일축으로 운용하는 구간입니다. | a/2 ≤ T ≤ a - 1 |
| 최초 매수시도액 | M0 | 사이클 시작 시점에 사용하는 첫 매수용 기준 금액입니다. 첫 체결 이후부터는 일반모드·리버스모드 매수시도액과 구분합니다. | M0 = P/a |
| 일반모드 1회매수시도액 | M | 일반모드에서 당일 주문 세트를 생성할 때 사용하는 기준 금액입니다. 잔금과 현재 T에 따라 매일 재계산됩니다. | M = C/(a - T) |
| 리버스모드 1회매수시도액 | M_rev | 리버스모드 둘째 날 이후 쿼터매수에 사용하는 기준 금액입니다. 해당 시점의 잔금 C의 1/4만 사용합니다. | M_rev = C / 4 |
| 보유수량 | H | 현재 계좌에 보유 중인 수량입니다. 매도 주문 수량, 리버스모드 매도 수량, 종료 판정의 기준입니다. | - |
| 줍줍매수 | J_n | 기본 주문세트 이후 가격 하락에 대응하기 위해 하단으로 1주씩 확장하는 예산 소진형 LOC 매수 사다리입니다. 6단은 예시형 제한이며, 원리는 동적 확장입니다. | J_n = M / (Q0 + n) |
| 기본 예상 체결 수량 | Q0 | 해당 모드의 기본 주문세트에서 예상 체결의 기준이 되는 수량입니다. 줍줍 가격 계산의 기준점으로 사용합니다. | 전반전: Q0 = q_s + q_a (잔돈 보정 포함) / 후반전: Q0 = q_s |
| 별지점 기본 주문 수량 | q_s | 별지점 LOC 기본축에서 계산되는 주문 수량입니다. 전반전에서는 1회매수시도액의 절반 금액, 후반전에서는 1회매수시도액 전체 금액을 기준으로 계산합니다. | 전반전: q_s = floor((M/2)/S_buy) / 후반전: q_s = floor(M/S_buy) |
| 평단가 기본 주문 수량 | q_a | 전반전에서 평단가 LOC 기본축에 배정되는 주문 수량입니다. 1회매수시도액의 절반 금액을 기준으로 계산합니다. | q_a = floor((M/2)/A) |
| 정규화 별지점 | S_norm | 별지점 원시 가격 S를 종목의 최소 호가단위에 맞춰 먼저 정규화한 가격입니다. 실주문 기준 별지점은 이 값을 사용합니다. | S_norm = normalize_price(S) |
| 정규화 함수 | normalize_price() | 내부 계산 가격을 실주문 가능한 가격 단위로 맞추는 함수입니다. 현재 기준으로는 주문가를 소수점 둘째자리(1센트)까지 반올림합니다. | normalize_price(x) = round(x, 2) |

2. V4.0을 일반화한 핵심 원리 (분할 수 무관 공식)

원문은 20분할과 40분할을 대표 예시로 설명하지만, 전략의 핵심 원리는 분할 수 a가 달라져도 동일 구조로 일반화할 수 있습니다. 아래는 원문 예시를 자동매매 관점에서 일반식으로 확장한 핵심 원리입니다.

<!-- table_4 -->
| 번호/원리 | 일반식·규칙 | 설명 |
| --- | --- | --- |
| ① 최초 매수시도액 | M0 = P / a | 새 사이클 시작일의 첫 매수 기준금액은 전체 원금을 전체 분할 수로 나눈 값입니다. |
| ② 일반모드 1회매수시도액 | M = C / (a - T) | 일반모드 둘째 날 이후 매수시도액은 잔금, 현재 T, 전체 분할 수에 의해 매일 재계산됩니다. |
| ③ 별% 일반식 | Star(T) = R × (1 - 2T/a) | 40분할·15%의 15 - 0.75T, 20분할·15%의 15 - 1.5T를 하나의 식으로 일반화한 형태입니다. |
| ④ 별지점 가격 | S = A × (1 + Star(T)/100)S_norm = normalize_price(S)S_buy = S_norm - 0.01S_sell = S_norm | 별지점 원시 가격 S를 먼저 계산한 뒤 호가단위에 맞춰 S_norm으로 정규화합니다. 매수 기준점은 S_norm - 0.01달러, 매도 기준점은 S_norm 그대로를 사용합니다. |
| ⑤ 모드 경계 | 전반전: T < a/2<br>후반전: a/2 ≤ T ≤ a – 1<br>리버스모드: T > a - 1 | 40분할이면 전반전 T < 20, 후반전 20 ≤ T ≤ 39, 리버스모드 T > 39와 연결됩니다. |
| ⑥ 일반모드 매수 원리 | 전반전: M의 절반을 별지점 LOC, 절반을 평단 LOC<br>후반전: M 전체를 별지점 LOC | 전반전은 2개의 축으로 진행되며, 후반전은 단일축으로 단순화됩니다. |
| ⑦ 기본 수량 일반식 | 전반전 q_s = floor((M/2)/(S_buy)), q_a = floor((M/2)/A)<br>후반전 q_s = floor(M/(S_buy)) | 전반전은 잔돈 합계로 추가 1주가 가능하면 더 낮은 가격 쪽에 1주를 우선 배정합니다. |
| ⑧ 줍줍매수 일반식 | J_n = M / (Q0 + n) | Q0는 전반전이면 q_s + q_a(잔돈 보정 포함), 후반전이면 q_s입니다. 아래 수식은 전체 분할 수를 a로 일반화한 식이며, 20분할·30분할·40분할 등 분할 수가 달라져도 동일 원리로 적용합니다. |
| ⑨ 매도 주문 일반 원리 | H 1/4 = 별지점 LOC 매<br>H의 나머지 3/4 = 평단 + 목표수익률 지정가 매도 | 수량 배분은 정수 기준으로 처리하되, 합계는 전체 보유수량과 일치해야 합니다. |
| ⑩ 일반모드 T 변화 | 1회 매수 체결: T + 1<br>절반 매수 체결: T + 0.5<br>쿼터매도 체결: 직전 T × 0.75 | 분할 수와 무관한 공통 이벤트 규칙입니다. |
| ⑪ 리버스모드 매도 수량 및 T 변화 | 리버스 매도 수량 = floor(H × 2/a)<br>매도 시 T = T × (1 - 2/a)<br>매수 시 T = T + (a - T) × 0.25 | 20분할은 1/10, 40분할은 1/20 매도와 정확히 연결됩니다. |
| ⑫ 리버스모드 매수시도액 | 첫날 매수 없음<br>둘째 날 이후 M_rev = C / 4 | C은 전일 체결 반영 후 실제 가용현금 전액으로 정의합니다. |
| ⑬ 큰수매수 일반 알고리즘 | 닫힌 수식 하나가 아니라 주문 재구성 알고리즘 | 현재가 대비 괴리가 큰 상단 매수 주문을 큰수 기준점으로 합산해 체결 가능성을 회복시키는 로직입니다. |

| --- | --- | --- | --- |
| 전반전 매수 | T < a/2 | 일반모드 1회매수시도액을 금액 기준 50:50으로 나누어, 절반은 별지점-0.01달러 LOC 매수, 나머지 절반은 평단가 LOC 매수로 배치합니다. 이후 줍줍매수 사다리를 생성합니다. | 정수 수량 내림 후 남는 잔돈 합계로 추가 1주 매수가 가능하면 더 낮은 가격 주문부터 1주씩 우선 배정합니다. |
| 후반전 매수 | a/2 ≤ T ≤ a - 1 | 일반모드 1회매수시도액 전체를 별지점-0.01달러 LOC 매수로 배치하고, 하단에는 줍줍매수 사다리를 생성합니다. | 평단 LOC는 제거합니다. 큰 하락·가격 괴리·주문 거부가 발생하면 큰수매수 전환을 허용합니다. |
| 매도 공통 | 일반모드 전후반전 공통 | 보유수량의 1/4은 별지점 LOC 매도, 나머지 3/4는 평단 대비 +목표수익률 지정가 매도로 배치합니다. | 수량은 정수 기준으로 배분하되, 두 매도 주문의 합계는 전체 보유수량과 정확히 일치해야 합니다. |
| 일반모드 T 판정 | 일반모드 체결 결과 발생 시 | 별지점 절반 체결 = +0.5, 평단 절반 체결 = +0.5, 둘 다 체결 = +1, 큰수매수 체결 = +1, 쿼터매도 체결 = ×0.75로 반영합니다. | 지정가 매도 후 같은 날 LOC 매수가 체결되면 먼저 ×0.75를 반영한 뒤 체결된 레그 수에 따라 +0.5 또는 +1을 더합니다. |

4. 큰수매수 알고리즘

큰수매수는 상단 체결 보장 압축 로직입니다. 새 사이클 시작 시에는 첫 진입 체결을 보장하기 위한 상단 주문으로 쓰이고, 사이클 중간에는 평단가·별지점 기준 주문이 실제 시세와 너무 벌어져 거부될 때 상단 주문을 합산해 체결 가능성을 회복시키는 대체 주문으로 사용합니다.

<!-- table_8 -->
| 구분 | 핵심 원리 | 구현 규칙 |
| --- | --- | --- |
| 시작 큰수 | 새 사이클 시작일의 첫 진입을 무조건 체결되게 하기 위한 상단 체결 보장 주문입니다. | 전일 종가 기준 상단 버퍼(+15% 기본값)를 사용하되, 브로커 허용 범위 안에서 정상 접수 가능한 가격으로 재산출합니다. |
| 중간 큰수 | 사이클 진행 중 별지점·평단가 기준 주문이 실제 시세와 지나치게 벌어져 주문 거부가 예상될 때 상단 매수 물량을 하나의 LOC로 압축합니다. | 기존 주문세트를 먼저 계산한 뒤 브로커 사전 검증 또는 접수 결과를 확인합니다. 상단 매수 주문이 실제 시세와 지나치게 벌어져 거부가 발생하거나 강하게 예상되면, 현재가보다 충분히 위이면서 별지점 아래에서 매수가 진행되도록 큰수 기준점을 정하고 그 기준점 이상 가격대의 기존 상단 매수 물량을 하나의 큰수 LOC로 합산합니다. |
| 매도 처리 | 큰수매수는 매수 측 재구성 로직입니다. | 매도 공통 주문세트는 원칙적으로 유지합니다. 다만 브로커가 매도 주문까지 거부하는 경우에는 무리한 대체 재발행보다 보수적 생략을 우선할 수 있습니다. |
| 버퍼 정책 | 원문 취지는 10%냐 15%냐보다 체결 보장입니다. | 기본 버퍼는 두되, 실제 구현에서는 장세·괴리·브로커 제한에 따라 +5%~+20% 범위에서 가변 선택할 수 있습니다. |

5. 줍줍매수 알고리즘

줍줍매수는 하단 예산 소진 확장 주문입니다. 6단 고정 주문이 아니라, 기본 주문세트 생성 후 해당 모드의 매수시도액을 가능한 한 추가 1주 주문으로 분해하여 소진하는 하단 사다리로 정의합니다.

<!-- table_9 -->
| 항목 | 원리 | 확정 구현 규칙 |
| --- | --- | --- |
| 실행 조건 | 첫 매수, 일반모드 전반전/후반전, 리버스모드 쿼터매수 생성 시 동일 원리로 적용할 수 있습니다. | 기본 동작은 동적 확장형으로 두고, 예시형 6단 제한은 선택 프로파일로 해석합니다. |
| 가격 계산 | 기본 예상 체결 수량을 Q0, 해당 모드의 매수시도액을 M이라고 하면, 줍줍 n단 가격은 M / (Q0 + n)으로 계산합니다. | 계산된 가격은 종목 틱 규칙에 맞게 먼저 호가단위로 반올림하여 정규화한 뒤 LOC 매수로 둡니다. |
| 생성 순서 | 기본 주문세트와 Q0를 먼저 계산한 뒤, n=1부터 순차적으로 하단 1주 주문가를 만듭니다. | 큰수매수로 상단 물량을 합산한 경우에는 합산 후의 Q0를 기준으로 다시 계산합니다. |
| 중단 조건 | 다음 1주 주문 생성이 불가능하거나, 운영 보호용 최대 주문 수에 도달하거나, 최대 하락폭을 넘거나, 동일 틱/최소 간격 제약에 걸리면 중단합니다. | 운영 보호용 상한은 전략 본질이 아니라 시스템 보호 장치로 취급합니다. |

8. 자동매매 적용 핵심 구현 메모

아래 구현 규칙은 실전 자동매매용으로 고정하는 추가 명세입니다. 원문 예시를 일반화하되, 무한매수법의 체결 우선·예산 최대 사용·모드 배타성 철학을 유지하도록 정리합니다.

<!-- table_12 -->
| 구현 항목 | 실전 구현 규칙 | 비고 |
| --- | --- | --- |
| ① 가격 산출과 반올림 규칙 | 모든 기준 가격은 내부 계산 후 먼저 종목의 최소 호가단위에 맞춰 반올림하여 정규화합니다. 현재 기준 정규화(normalize_price)는 주문가를 소수점 둘째자리(1센트)까지 반올림하는 것을 뜻합니다. 별지점은 S = A × (1 + Star(T)/100)로 계산한 뒤 S_norm = normalize_price(S)로 정규화하며, 매수용 별지점은 S_buy = S_norm - 0.01, 매도용 별지점은 S_sell = S_norm으로 사용합니다. 줍줍매수 가격 J_n도 내부 계산 후 같은 방식으로 정규화합니다. | 별지점 매수점은 정규화 이후 0.01달러를 차감합니다. |
| ② 수량 정수화와 잔돈 보정 규칙 | 전반전 기본 매수는 일반모드 1회매수시도액 M을 금액 기준 50:50으로 나누어 q_s = floor((M/2)/S_buy), q_a = floor((M/2)/A)로 계산합니다. 그 다음 두 레그의 잔돈을 합산하여 추가 1주 매수가 가능하면 더 낮은 가격 주문부터 1주씩 우선 배정하여 1회매수시도액을 최대한 사용합니다. 이 잔돈 보정은 전반전 기본축(q_s, q_a)에만 적용하고, 줍줍매수는 별도 사다리로 항상 생성합니다. 매도 수량은 quarter_qty를 먼저 정하고 final_qty = H - quarter_qty로 계산하여 두 수량의 합이 반드시 H와 일치하게 합니다. | 매수 수량은 floor 고정, 매도는 quarter_qty 선결정 후 나머지를 final_qty에 배정합니다. |
| ③ 큰수매수 발동 조건과 재구성 규칙 | 큰수매수는 기존 상단 매수 주문이 실제 시세와 지나치게 벌어져 주문 거부가 발생하거나 강하게 예상되어 별지점 아래에서 매수가 정상 진행되지 못할 때 발동하는 상단 체결 보장 압축 로직으로 구현합니다. 시작 큰수는 첫 진입 체결 보장용, 중간 큰수는 진행 중 상단 매수 물량을 하나의 LOC로 합산하는 재구성 로직으로 구분합니다. 발동 시에는 현재 모드의 기본 주문세트를 먼저 계산한 뒤, 큰수 기준점 이상 가격대의 기존 상단 매수 물량만 하나의 큰수 LOC로 합산하고 그 아래 하단 1주 사다리는 유지합니다. | 큰수 기준점은 현재가보다 충분히 위이면서 별지점 아래에서 매수가 진행되도록 선택합니다. |
| ④ 실전 주문 생성·발행·정리 순서 | 실전 주문은 주간장을 사용하지 않고 전일 정규장 종료 후 확정된 상태를 기준으로 다음 거래일 주문세트를 생성합니다. 매일의 처리 순서는 전일 체결 확정 → 상태(T, C, H, A) 갱신 → 모드 판정 → 당일 주문세트 계산 → 브로커 사전 검증(precheck) → 필요 시 큰수매수 재구성 → 주문 발행으로 고정합니다. 일반모드에서는 매수는 모두 LOC로 생성하고, 매도는 1/4 별지점 LOC + 3/4 평단 + R 지정가로 생성합니다. 지정가 매도는 프리장 시작 시점에 발행하여 프리장·본장·애프터까지 유지하고, LOC 매수·매도는 브로커가 허용하는 예약 또는 정규장 제출 방식 중 하나로 일관되게 운용합니다. 당일 주문세트는 하루에 하나의 모드만 존재해야 하며, 전일 미체결 잔량은 다음 주문 생성 직전에 취소 후 전량 재계산합니다. 리버스모드 첫날은 예외적으로 MOC 매도만 발행하고 매수는 발행하지 않습니다. | 브로커 adapter가 LOC/MOC/지정가의 실제 API 매핑을 책임집니다. |
| ⑤ 분할 수 일반화 적용 원칙 | 아래 일반식은 전체 분할 수를 a로 일반화한 식이며, 20분할·30분할·40분할 등 분할 수가 달라져도 동일 원리로 적용합니다. 따라서 전략 엔진은 특정 분할 수를 하드코딩하지 않고, M = C / (a - T), Star(T) = R × (1 - 2T/a), 전반전·후반전·리버스모드 경계, 리버스모드 매도 수량 floor(H × 2/a), 리버스모드 T 변화식을 모두 같은 일반식 함수로 처리합니다. | 분할 수 설명은 별도 장황하게 반복하지 않고, 일반식이 스스로 원리를 설명하도록 둡니다. |
| ⑥ 미체결·부분체결 재계산 원칙 | 전일 주문 중 미체결·부분체결·만료 주문은 다음 거래일 주문 생성 직전에 먼저 조회하여 상태를 확정합니다. 미체결 잔량을 그대로 다음 날로 들고 가지 않고, 취소 가능 주문은 취소한 뒤 전일 최종 체결 결과를 반영하여 당일 주문세트를 처음부터 다시 계산합니다. 부분체결도 잔량 유지 방식이 아니라 체결분을 상태(T, C, H, A)에 반영한 뒤 잔량 포함 전체 주문세트를 재구성합니다. 리버스모드에서도 같은 원칙을 적용합니다. | 잔량 유지보다 상태 재계산을 우선합니다. 동일 사이클이라도 전일 체결 결과가 달라지면 주문세트를 새로 계산합니다. |
| ⑦ 중복 발행 방지와 주문세트 식별 규칙 | 당일 주문세트에는 반드시 전략버전, 종목, 사이클 ID, 거래일, 모드, 주문다리 번호를 포함한 고유 식별키를 부여합니다. 주문 발행 전에는 동일 식별키의 기존 접수·예약 주문이 있는지 먼저 확인하고, 이미 존재하면 재발행하지 않습니다. 브로커 응답 지연이나 재시도 상황에서도 주문세트 기준 멱등성을 유지하여 같은 주문이 중복 접수되지 않도록 합니다. | 클라이언트 주문키·주문세트 ID는 저장소와 로그에 함께 남겨 사후 추적 가능해야 합니다. |
| ⑧ 안전정지·수동개입 전환 규칙 | 아래 상황에서는 자동 진행보다 안전정지를 우선합니다: 브로커 사전 검증 실패가 반복되는 경우, 주문 거부가 연속 발생하는 경우, 기준 가격·별지점·리버스 기준 데이터가 비어 있거나 상충하는 경우, 보유수량·평단·잔금이 브로커 잔고와 일치하지 않는 경우, 거래정지·호가 공백·API 장애로 주문세트 유효성을 보장할 수 없는 경우입니다. 안전정지 시에는 신규 주문 발행을 중단하고, 운영자 확인 후 수동개입 또는 복구 승인 절차를 거쳐야만 자동매매를 재개합니다. | 리버스모드 판단 데이터 누락은 즉시 안전정지 사유로 처리합니다. |
| ⑨ 재시작 복구와 리버스모드 동기화 규칙 | 프로그램 재시작 시에는 마지막 저장 상태를 그대로 신뢰하지 않고, 브로커 체결·잔고·미체결 주문을 다시 조회하여 사이클 상태를 재동기화한 뒤에만 주문 생성을 허용합니다. 특히 V4.0은 일반모드와 리버스모드가 배타적이므로, 재시작 복구 단계에서 현재 모드, 이어받은 T, 보유수량 H, 평단 A, 잔금 C, 리버스모드 전용 기준값과 당일 발행 예정 주문세트가 서로 일치하는지 검증해야 합니다. 하나라도 불일치하면 자동 복구하지 않고 수동 확인 상태로 전환합니다. | 본 항목은 실전 자동매매 안정성을 위한 설계 판단입니다. |
