🚨 제1헌법 [ 동기 I/O 100% 비동기 격리 ] : 모든 파일 읽기/쓰기(open, json.load/dump), 달력 API(mcal), 외부 통신은 예외 없이 asyncio.to_thread로 백그라운드 스레드에 밀어내고, 무한 대기를 막기 위해 타임아웃(wait_for) 족쇄를 채워 메인 이벤트 루프의 교착(Deadlock)을 원천 차단하라.
🚨 제2헌법 [ 단일 책임 원칙과 중복 코드 소각 ] : 하나의 파일 내에 동일한 함수명이 중복 선언되는 프랑켄슈타인 덮어쓰기 사고를 절대 금지한다. 정적 분석기(Linter)의 엄격한 관점으로 사용되지 않는 변수, 연결되지 않은 데드코드, 낡은 로직은 발견 즉시 영구 소각하라.
🚨 제3헌법 [ 타임존 단일 소스 락온 (EST 100%) ] : 시스템의 뼈대가 되는 모든 논리적 시계열(스냅샷, 장부, 스케줄러 기준점)은 오직 미국 동부 시간(ZoneInfo('America/New_York'))으로만 100% 락온한다. 한국 시간(KST)은 오직 텔레그램 UI 렌더링(출력) 용도로만 철저히 격리하여 타임 패러독스를 원천 봉쇄하라.
🚨 제4헌법 [ 예측 가능한 멱등성 및 원자적 쓰기 ] : 장부 및 상태 캐시 파일 보존 시 반드시 임시 파일 생성 후 덮어쓰는 원자적 쓰기(Atomic Write)와 비동기 래핑을 강제하여, 서버가 어느 타이밍에 다운되더라도 파일 파손이나 반쪽짜리 상태(Torn Write)가 발생하지 않도록 멱등성을 절대 보장하라.
🚨 제5헌법 [ 엣지 타임라인 Fail-Safe 구조화 ] : 장마감 직전(15:57 EST 등) 특정 시간에만 격발되는 치명적 로직은 달력 API 지연이나 시간 오차(Jitter)에 봇이 기절하지 않도록 반드시 안전 폴백(Fallback, 예: 평일 강제 개장) 경로를 하드코딩하여 시스템의 런타임 붕괴를 영구 차단하라.
🚨 [ 치명적 경고 1: 타임존 락온(Lock-on) 및 스냅샷 세션 간 오염 절대 차단 ] :
모든 날짜 연산(스냅샷, 캐시, 장부, API 체결 조회) 시 제3헌법에 의거 datetime.now(ZoneInfo('America/New_York')).strftime("%Y-%m-%d") 포맷을 강제 적용하여 봇의 논리적 시계를 미국 동부 시간(EST/EDT)에 100% 종속시키세요.
향후 신규 모듈이나 스케줄러가 추가되더라도 예외 없이 EST 락온을 걸어야 하며, KST를 Fallback으로 사용하면 아직 열리지 않은 '미래의 미국 날짜'를 조회하게 되어 스냅샷 매핑이 증발하는 치명적 맹점(EC-1, EC-2, EC-3)이 발생합니다.
또한 APScheduler 잡(Job) 등록 시, KST 기반의 동적 연산(TARGET_HOUR 등)을 전면 소각하고 오직 미국 동부 시간(EST/EDT)을 tzinfo=ZoneInfo('America/New_York')로 고정하여 멱등하게 주입해야 합니다. 이를 위반하면 타임존 패러독스로 인해 매매 초기화 및 통합 주문 스케줄 자체가 큐에서 증발(Skip)하는 치명적 버그가 발생합니다.
또한, 17:05 KST에 생성된 당일 지시서 스냅샷(daily_snapshot_*.json)은 익일 정기 초기화 전까지 어떠한 경우에도 물리적으로 숨기거나 삭제해선 안 되며, 내일 날짜로 미리 복사(shutil.copy)하는 레거시 코드를 절대 부활시키지 마세요. 당일 스냅샷은 오직 당일 날짜(EST 기준)로 단 1회만 멱등하게(Idempotent) 생성되어야 합니다.
🚨 [ 치명적 경고 2: 텔레그램 수동 조작 시 다이렉트 파일 I/O 비동기 래핑 및 멱등성 보장 ] :

🚨 [ 치명적 경고 3: VWAP 실시간 타격 vs 텔레그램 지시서(UI) 디커플링 절대 유지 ] :

[ 5대 핵심 교전 룰 락온 ]
 1. 타임라인 및 무한 스캔 개방: 10:00 EST 단판 승부 족쇄를 전면 폐기하고, 프리마켓 개장(04:00 EST)부터 14:59 EST까지 실시간 무한 레이더 감시를 집행합니다. 옴니 매트릭스의 pending 락다운 쉴드를 바이패스(Bypass)하여 04:00부터 타격을 100% 허용합니다.
 2. 하이킨아시 3대 조건 팩트 스캔: 1) 원웨이 (롱/숏 당일 고저가 방향 일치). 2) 하이킨아시 모멘텀 (실시간 VWAP 방향 일치 및 5분 리샘플링 HA '꼬리 없는 캔들' 2연속 출현, 단 0.01$ 이하 오차는 꼬리 없음으로 수용). 3) 당일 저가 기준 ATR5 잔여 체력 1.0% 이상. 이 3가지 100% 충족 시만 즉각 타격.
 3. 투트랙 엑시트: AUTO 모드는 하이킨아시 2연속 역추세(롱: 윗꼬리 없는 음봉 2회, 숏: 아래꼬리 없는 양봉 2회) 발생 시 즉각 덤핑. MANUAL 모드는 사용자 설정 고정 목표 수익률 도달 시 지정가 즉각 청산. 1회 익절 후에도 조건 재충족 시 무한 재진입(다중 출장)을 100% 허용합니다.
 4. 50% 동적 예산 엔진: 본대(V-REV)의 15% 가상 에스크로(Virtual Escrow) 보호막을 시스템 전역에서 완전히 소각하고, 암살자 출격 시 KIS 실시간 순수 주문 가능 금액(cash)의 50%를 즉시 할당받습니다. 수수료 및 슬리피지 거절 방지를 위해 산출 예산의 95%만 투입합니다.
 5. 절대 존버(Hold) 및 15:00 타임스탑: 하드스탑(-8.0%) 전면 폐기(절대 손절 금지). 15:00 EST 도달 시 신규 진입을 무조건 차단합니다. 포지션 보유 중 수익 상태면 즉각 전량 팩트 덤핑 퇴근, 손실 상태면 매도를 중단하고 오버나이트 존버(홀딩) 궤도로 영구 동결합니다.
🚨 [ 치명적 경고 14: 당일 제로섬(Zero-Sum) 매매 시 장부 누락 및 데이터 기아 방어 ] :
장중 수동 매수 후 수동으로 전량 익절하여 실잔고 오차가 0주(Zero-Sum)가 되었을 때, 메인 장부(Ledger) 업데이트가 이를 스킵하여 체결 영수증이 증발하는 맹점을 막기 위해 needs_reconstruction 팩트 스캔 로직(당일 원장 매수/매도 수량 대조)을 절대 유지하세요. 또한 KST 자정 크로스오버 시 원장이 누락되는 데이터 기아(Starvation) 현상을 막기 위해 과거 4일 치를 광역 스캔 후 EST 타임존으로 정밀 형변환(filter_to_est)하는 엔진을 훼손하지 마세요.
🚨 [ 치명적 경고 15: V-REV 가상 에스크로 영구 철거 및 암살자 50% 예산 비동기 락온 ] :
V-REV 모드에서 자전거래(Wash Trade) 의심을 피하기 위해 물리적인 예방적 덫(LOC) 전송을 전면 소각했습니다. 암살자(AVWAP) 출격 예산을 계좌 가용 현금의 50%로 확장함에 따라, 본대(V-REV/V14) 매수 증거금과 충돌(API Reject)이 발생하지 않도록 비동기 락(tx_lock)을 철저히 검증하고 예산 할당량의 멱등성을 100% 보장하십시오.
🚨 [ 치명적 경고 16: 스케줄러 달력 API 타임아웃 방어 및 Fail-Open 락온 ] :
제5헌법에 따라 정규장, VWAP, 스나이퍼 등 모든 스케줄러 진입 시 장 개장 여부를 묻는 is_market_open 류의 동기 함수는 무조건 비동기 타임아웃(10초)으로 감싸야 하며, 타임아웃 발생 시 스케줄이 허공에 증발하는 것을 막기 위해 '오늘이 평일(월~금)이면 무조건 강제 개장(Fail-Open)'으로 간주하는 안전 폴백(Fallback) 방어막을 필수적으로 하드코딩하세요.
🚨 [ 치명적 경고 17: 좀비 프로세스 및 시스템 킬(Kill) 레이스 컨디션 절대 방어 ] :
plugin_updater.py에서 데몬 재가동 시 OS 쉘을 통한 sudo systemctl restart 명령어에 절대 의존하지 마세요. 텔레그램 우아한 셧다운 대기 중 백그라운드 스케줄러(sleep 등)와 충돌하여 봇이 무한 대기하는 좀비(Zombie) 상태에 빠집니다. 업데이트 동기화 직후에는 반드시 파이썬 자체 하드 킬(os._exit(0))을 격발시켜 즉각 자폭하고, OS(systemd)의 Restart=always 속성이 이를 예비 가동시키도록 무중단 아키텍처를 절대 사수하세요.
🚨 [ 치명적 경고 18: 변수 스코프(Scope) 전진 배치 및 UnboundLocalError 원천 봉쇄 ] :
조건문(if/else) 내부에서 변수를 최초 선언(초기화)한 뒤, 조건문 외부 하단에서 해당 변수를 참조하는 논리적 맹점을 절대 엄금합니다. (예: target_sweep_qty, sweep_type 등)
0주 새출발과 같은 특정 엣지 케이스에서 조건문을 패스할 경우, 변수가 메모리에 아예 생성되지 않아 파이썬 코어 엔진이 UnboundLocalError를 뱉고 스케줄러 루프 전체가 즉사(기절)하는 치명적 데이터 기아(Data Starvation) 현상이 발생합니다.
모든 참조 변수는 반드시 로직의 최상단(조건문 진입 전)에서 기본값(0, 0.0, "" 등)으로 무조건 선언(Scope Lift)하여 런타임 붕괴의 싹을 영구 소각하세요.
🚨 [ 코파일럿 리포트 교차 검증 및 역지시어 생성 엔진 가동 ]
이후 대화창에 입력되는 텍스트 중 코파일럿의 분석 리포트가 감지되면, 즉시 이를 해부하여 시스템에 탑재된 [방탄 아키텍처 5대 절대 헌법 및 절대 방어 엣지 케이스 18계명]과 대조하는 교차 검증 파이프라인을 가동하라.
🔹 1단계 (전면 소각 및 백신 주석 이식) : 나의 코어 룰(KST 자정 패러독스 유발, 큐 장부 통째로 덮어쓰기, VWAP 디커플링 파괴, 이중 합산, 동기 블로킹 등)을 우회/훼손하거나, 시스템의 전체 아키텍처를 이해하지 못해 발생한 코파일럿의 '환각(Hallucination) 엣지 케이스' 제안은 전면 소각(기각)하라. 기각 사유를 건조하게 브리핑함과 동시에, 후속 AI 에이전트들이 동일한 코드를 읽고 같은 오판을 반복하지 않도록 타겟 코드 심장부에 🚨 [AI 에이전트(Copilot/Claude) 절대 주의 - 환각(Hallucination) 방어막] 이라는 명시적인 백신 주석(경고문)을 하드코딩하여 삽입하는 수술을 즉각 병행하라.
🔹 2단계 (제한적 수용) : 코어 룰을 침범하지 않으면서 런타임 붕괴를 막아주는 유효한 방어 로직(예외 처리, 멱등성 강화 등)만 핀셋으로 수용하라.
🔹 3단계 (역지시어 출력) : 필터링이 완료되면, 승승장군이 IDE 대화창에 그대로 복사하여 코파일럿에게 하달할 수 있는 '방탄 리팩토링 명령 프롬프트'를 최종 출력하라. 이 역지시서에는 코파일럿이 코어 룰을 건드리지 않고 안전하게 수정된 코드만 렌더링하도록 락온(Lock-on)하는 통제 가이드가 포함되어야 한다.
